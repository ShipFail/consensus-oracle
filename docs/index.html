How Priorscope works

Under the hood, Priorscope calls multiple models with deterministic decoding and uses embeddings to compute how aligned their answers are.

Designed as a low-friction API and dashboard you can drop into eval pipelines, agent frameworks, or research experiments.

1

Ask one question

Submit a short, well-posed question via the web UI or API. Priorscope normalizes and logs it for reproducible experiments.

2

Query multiple models

We call Gemini, GPT, Claude (and optionally open-source models) with temperature 0 and greedy decoding to remove randomness and expose the underlying priors.

3

Compute consensus score

Answers are embedded into a shared vector space; Priorscope calculates a consensus score and surfaces agreements, disagreements, and edge cases.

Questions & caveats

Priorscope is a tool for measuring consensus, not a final arbiter of truth. Here’s how we think about that.

Does high consensus guarantee truth?

No. It simply indicates that multiple models, trained on overlapping human data, strongly agree. That usually correlates with stable, widely accepted knowledge — but they can still agree on a shared hallucination.

What does low consensus tell me?

Low consensus often marks ambiguity: poorly posed questions, controversial topics, under-specified tasks, or areas where models have different training or alignment. Great places to add human review.

Do you call provider APIs directly?

For the hackathon edition, you plug in your own API keys for providers like OpenAI, Google, and Anthropic. We never resell model access; we just orchestrate deterministic calls.

Is Priorscope production-ready?

Today it’s a fast-moving MVP aimed at hackathons, internal tools, and research workflows. If you’re exploring production integration, we’d love to chat.
